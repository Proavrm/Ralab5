RaLab - Référentiel métier du service scientifique, technique et laboratoire
Flux opérationnel complet : affaires, demandes, campagnes, interventions, essais, G3, fiches de suivi et saisie
Version de travail

====================================================================
1. OBJET DU DOCUMENT
====================================================================

Ce document sert de reference metier structurante pour la suite du developpement de RaLab.
Il decrit le fonctionnement cible d'un service scientifique, technique et laboratoire intervenant :
- en laboratoire,
- sur chantier,
- en reconnaissance geotechnique,
- en controles d'execution,
- en diagnostics,
- en appui technique,
- en mission G3,
- en interpretation et restitution technique.

L'objectif n'est pas seulement de decrire un laboratoire d'essais classique, mais un service complet capable de transformer une demande terrain, chantier, etude ou client en action technique planifiee, traçable, interpretable et exploitable.

Ce document doit aider a structurer :
- le modele metier,
- les objets de la base de donnees,
- les ecrans,
- les statuts,
- les relations,
- les fiches de suivi,
- les formulaires de saisie,
- les resultats a afficher,
- les livrables,
- la logique de validation,
- la traçabilite.

====================================================================
2. VISION GENERALE DU FLUX METIER
====================================================================

Le flux metier cible est le suivant :

Affaire
-> Demande
-> Campagne
-> Preparation de l'intervention
-> Intervention
-> Point / prelevement / sondage / controle / observation
-> Echantillon
-> Traitement laboratoire
   -> Preparation d'echantillon
   -> Essais
-> Validation technique
-> Resultats / PV / note / rapport / avis technique
-> Facturation / archivage / retour d'experience

Points de vigilance importants :

1. La campagne est un objet metier central. Elle se situe entre la demande et les interventions.
2. La preparation de l'intervention doit etre distinguee de la preparation d'echantillon.
3. Le service ne se limite pas aux essais. Il couvre aussi l'appui technique et les missions G3.
4. Le systeme doit permettre une restitution factuelle, mais aussi une interpretation technique.
5. Les workflows doivent rester utilisables en laboratoire pur, en terrain pur, et en mixte terrain + laboratoire + interpretation.

====================================================================
3. DEFINITION DES OBJETS METIER
====================================================================

3.1 Affaire

L'affaire est le cadre global du dossier.
Elle correspond au contexte contractuel, operationnel et financier dans lequel s'inscrivent les demandes.

Informations a rattacher a l'affaire :
- reference affaire,
- client,
- entreprise responsable / entite interne,
- chantier,
- site,
- localisation,
- type d'operation,
- contexte contractuel,
- devis / commande / accord interne,
- budget,
- contacts,
- documents de reference,
- responsables,
- statut global,
- contraintes specifiques,
- dates clefs,
- lien avec dossiers techniques existants.

3.2 Demande

La demande est le besoin technique exprime au service scientifique, technique et laboratoire.
Elle peut concerner :
- des essais de laboratoire,
- des controles terrain,
- des prelevements,
- des reconnaissances geotechniques,
- des controles d'execution,
- des prestations d'appui technique,
- des expertises,
- des diagnostics,
- des missions G3,
- des avis techniques,
- des complements d'investigation.

Exemples de demandes :
- faire une campagne geotechnique,
- realiser des essais de plaque sur plateforme,
- faire des prelevements de GNT et essais de conformite,
- realiser des carottages enrobes,
- faire des essais d'identification sur sols,
- verifier la compacite de remblais,
- controler un beton frais,
- faire une reconnaissance de fondation,
- lancer une expertise suite a non-conformite,
- realiser un appui geotechnique en phase execution,
- definir et suivre une mission G3,
- proposer des reconnaissances complementaires dans le cadre d'une G3,
- interpreter les resultats d'execution et emettre un avis technique,
- suivre la conformite geotechnique d'une plateforme, d'un terrassement, d'une couche de forme ou d'un soutenement,
- verifier un reemploi de materiaux,
- analyser une pathologie de chaussee,
- realiser un suivi d'execution de type G3 pour terrassements, fondations, couche de forme ou soutenement.

3.3 Campagne

La campagne est l'objet de pilotage operationnel.
Elle permet d'organiser la reponse a une demande par zone, phase, periode, lot, objectif ou programme technique.

La campagne n'est ni une simple etiquette, ni une simple vue de regroupement.
C'est un vrai niveau metier permettant de :
- definir un objectif technique,
- structurer le programme d'action,
- planifier,
- affecter les moyens,
- regrouper les interventions,
- suivre l'avancement,
- consolider les resultats,
- produire une synthese.

3.4 Preparation de l'intervention

La preparation de l'intervention couvre toutes les actions realisees avant execution terrain ou laboratoire operationnel.
Elle comprend :
- analyse technique du besoin,
- choix du type d'intervention,
- definition des points,
- planification,
- affectation des ressources,
- verification des habilitations,
- verification du materiel,
- verification metrologique,
- preparation documentaire,
- coordination chantier,
- verification des acces,
- anticipation HSE,
- validation technique si necessaire.

3.5 Intervention

L'intervention est l'action concrete planifiee et executee.
Exemples :
- visite chantier,
- essai de plaque,
- carottage,
- prelevement,
- sondage,
- controle beton frais,
- recontrole,
- contre-visite,
- pose ou releve de materiel,
- visite G3,
- reunion technique sur site,
- visite de constat,
- campagne de description geotechnique.

3.6 Point / prelevement / sondage / controle / observation

Il s'agit du niveau terrain fin.
Une intervention peut contenir plusieurs points.
Selon le cas, un point peut etre :
- un point d'essai,
- un point de prelevement,
- un sondage,
- un carottage,
- un point d'observation,
- un point de controle,
- un point de mesure,
- une fouille / un puits,
- une zone de constat.

3.7 Echantillon

L'echantillon est l'objet matiere et traçabilite.
Il peut provenir :
- d'un prelevement chantier,
- d'un sondage,
- d'un carottage,
- d'une livraison au laboratoire,
- d'un controle production,
- d'un depot direct par un technicien ou demandeur,
- d'un doublon ou fractionnement d'un echantillon parent.

3.8 Traitement laboratoire

Le traitement laboratoire regroupe ce qui est fait sur l'echantillon en laboratoire.
Il comprend :
- la reception,
- l'enregistrement,
- le stockage,
- la preparation d'echantillon,
- les essais,
- le calcul,
- la validation,
- la restitution des resultats.

3.9 Preparation d'echantillon

La preparation d'echantillon est distincte de la preparation de l'intervention.
Elle correspond aux operations physiques realisees sur un echantillon avant ou pendant les essais.
Exemples :
- sechage,
- quartage,
- reduction,
- desagregation,
- tamisage preparatoire,
- lavage,
- homogeneïsation,
- sciage,
- decoupe,
- confection d'eprouvettes,
- cure,
- remoulage,
- conditionnement.

3.10 Essai

L'essai est l'operation technique de mesure, controle, caracterisation ou verification.
Il peut etre :
- in situ,
- en laboratoire,
- interne,
- sous-traite,
- simple,
- calcule,
- compose,
- interprete.

3.11 Validation technique

La validation technique couvre :
- verification de coherence,
- controle des valeurs,
- verification des calculs,
- verification des unites,
- verification du respect de la methode,
- visa par personne habilitee,
- emission ou blocage.

3.12 Restitution

La restitution peut prendre plusieurs formes :
- fiche terrain,
- compte rendu d'intervention,
- PV d'essai,
- synthese de campagne,
- rapport de reconnaissance,
- note d'analyse,
- avis technique,
- note G3,
- alerte non-conformite,
- tableau de suivi,
- rapport final.

4. RELATIONS ENTRE OBJETS
====================================================================

La logique cible doit permettre au minimum :
- une affaire peut contenir 0..n demandes,
- une demande appartient a 1 affaire,
- une demande peut contenir 0..n campagnes,
- une campagne appartient a 1 demande,
- une campagne peut contenir 0..n preparations de l'intervention,
- une campagne peut contenir 0..n interventions,
- une intervention appartient a 1 campagne,
- une intervention peut contenir 0..n points,
- un point appartient a 1 intervention,
- un point peut produire 0..n echantillons,
- un echantillon appartient a 0..1 point ou peut etre cree par reception directe,
- un echantillon peut avoir 0..n traitements labo,
- un traitement labo appartient a 1 echantillon,
- un traitement labo peut contenir 0..n operations de preparation d'echantillon,
- un traitement labo peut contenir 0..n essais,
- un essai appartient a 1 traitement labo,
- un essai peut produire 0..n resultats,
- une campagne peut etre liee a 0..n livrables,
- une demande peut etre liee a 0..n livrables,
- une affaire peut consolider 0..n livrables.

5. GRANDES PHASES DU PROCESSUS METIER
====================================================================

5.1 Prise d'affaire

Objectif : comprendre le besoin, qualifier la prestation, cadrer le perimetre, verifier la faisabilite.

Contenu attendu :
- reception de la sollicitation,
- identification du demandeur,
- qualification technique initiale,
- lecture rapide des pieces disponibles,
- identification du contexte contractuel,
- analyse du besoin reel,
- estimation des moyens,
- evaluation des risques,
- verification des delais,
- chiffrage,
- validation ou refus,
- ouverture du dossier.

5.2 Revue technique de la demande

Objectif : transformer le besoin exprime en programme technique clair.

Contenu attendu :
- lecture des documents,
- identification des questions techniques,
- determination des hypotheses,
- choix des types d'investigation,
- determination des zones et frequences,
- determination des types d'essais,
- identification des risques,
- identification des limites de mission,
- qualification du niveau d'urgence,
- besoin de sous-traitance ou non,
- choix des livrables.

5.3 Construction de la campagne

Objectif : organiser la reponse a la demande en un programme coherent et pilotable.

Contenu attendu :
- type de campagne,
- objectif,
- zone / secteur / phase / lot,
- temporalite,
- interventions prevues,
- equipe,
- ressources,
- materiels,
- contraintes HSE,
- dependances chantier,
- criteres de cloture,
- documents de reference,
- livrables attendus.

5.4 Preparation de l'intervention

Objectif : rendre l'execution possible, sure, efficace et tracee.

Contenu attendu :
- ordre de mission,
- affectation equipe,
- verification qualification,
- verification metrologie,
- verification disponibilite materiels,
- preparation EPI,
- preparation consommables,
- preparation fiches terrain,
- plan d'acces,
- contacts chantier,
- localisation points,
- verification autorisations,
- verification plan de prevention / PPSPS si besoin,
- verification logistique,
- coordination horaires,
- anticipation coactivite,
- confirmation du programme.

5.5 Realisation de l'intervention

Objectif : executer l'action technique sur le terrain ou en contexte d'exploitation.

Contenu attendu :
- accueil chantier,
- consignes securite,
- verification conditions reelles,
- adaptation si necessaire,
- realisation des points,
- observations,
- mesures,
- prelevements,
- photos,
- localisation,
- incidents / ecarts,
- synthese de fin d'intervention,
- transfert des echantillons,
- information immediate en cas d'anomalie critique.

5.6 Traitement laboratoire

Objectif : assurer un traitement trace, fiable et conforme des echantillons.

Contenu attendu :
- reception,
- enregistrement,
- verification de conformite a reception,
- stockage,
- preparation d'echantillon,
- realisation essais,
- calculs,
- validation,
- emission des resultats.

5.7 Interpretation technique et restitution

Objectif : transformer les resultats en information exploitable.

Contenu attendu :
- verification coherence,
- comparaison aux criteres,
- interpretation,
- decision de conformite,
- recommandations,
- besoins complementaires,
- rapport / note / PV,
- diffusion,
- archivage,
- retour d'experience.

13. FICHES DE SUIVI - PRINCIPES GENERAUX
====================================================================

Une fiche de suivi doit servir a piloter, tracer et verifier l'execution d'une activite.
Elle n'est pas seulement une feuille de notes.
Elle doit permettre :

- la comprehension rapide,
- le suivi d'avancement,
- la verification de completude,
- la preuve d'execution,
- la validation,
- l'archivage,
- la transmission.

Chaque fiche de suivi doit idealement comporter 4 blocs :
1. Identification
2. Preparation / contexte
3. Execution / suivi

44. PRINCIPES STRUCTURANTS POUR RALAB
====================================================================

1. Ne pas modeliser le service comme un simple catalogue d'essais.
2. Maintenir la distinction entre demande, campagne et intervention.
3. Integrer nativement la logique terrain + labo + interpretation.
4. Integrer nativement la logique G3 / appui technique.
5. Distinguer preparation de l'intervention et preparation d'echantillon.
6. Regrouper preparation d'echantillon + essais dans un bloc traitement laboratoire.
7. Permettre un usage simple pour l'operateur, sans perdre la finesse metier.
8. Permettre des workflows incomplets ou progressifs sans casser la traçabilite.
9. Permettre la saisie rapide terrain et la validation differée.
10. Permettre la restitution factuelle et la restitution interpretee.
11. Permettre l'historisation, les reprises, les complements et les avis successifs.
12. Faire du logiciel un outil d'aide a la decision technique, pas seulement un registre de valeurs.

Le service cible de RaLab doit etre compris comme un service scientifique, technique, laboratoire et geotechnique complet.
Il ne doit pas etre limite a la production de resultats d'essais.
Il doit permettre :
- de recevoir une demande,
- de la qualifier,

Ce document doit servir de socle metier pour la suite du developpement RaLab.

46. MATRICE DE RATTACHEMENT METIER DES ESSAIS
====================================================================

Important :
Le bon raisonnement n'est pas seulement par support final de l'essai.
Le bon raisonnement est par chaine metier complete.

La base commune reste :
Affaire -> Demande -> Campagne -> Intervention

Ensuite, selon le type d'essai, la chaine continue via :
- point terrain,
- ouvrage / troncon,
- prelevement,
- echantillon,
- lot / fabrication,
- fiche de synthese.

Rappels de logique :
- un PMT ne doit pas etre rattache directement a un echantillon ; il passe par un point terrain,
- une granulometrie de sol ne doit pas etre rattachee directement a l'intervention ; elle passe par prelevement puis echantillon,
- un IM doit suivre la chaine : Affaire -> Demande -> Campagne -> Intervention -> Prelevement -> Echantillon -> Essai IM,
- une fiche ID n'est pas un essai brut ; c'est une synthese d'essais sources,
- un essai de production ou de fabrication doit pouvoir passer par un objet lot / fabrication.

Colonnes de lecture du tableau ci-dessous :
- Code = code metier ou code d'essai,
- Libelle metier = nom operationnel a retenir,
- Famille = famille technique,
- Nature = terrain / laboratoire / production / synthese,
- Chaine metier complete = sequence logique complete avant l'essai,
- Support final de l'essai = objet parent direct pertinent pour l'essai,
- Sortie principale = type de restitution principal.

- un essai ne se cree pas toujours depuis le meme parent,

- mais la chaine amont reste ordonnee et commune : Affaire -> Demande -> Campagne -> Intervention,
- ensuite on bifurque selon le cas vers point_terrain, ouvrage_troncon, prelevement, echantillon, lot_fabrication ou fiche_synthese.

RÈGLES DE CONCEPTION À CONSERVER

1. Tous les essais partent d'une chaîne métier commune :
Affaire → Demande → Campagne → Intervention

2. L'essai ne doit pas toujours être créé directement sous l'intervention.
Selon le type d'essai, il faut créer d'abord l'objet intermédiaire pertinent :
- point terrain
- ouvrage / tronçon
- prélèvement
- échantillon
- lot / fabrication
- fiche de synthèse

3. Parent direct de l'essai :
- essais terrain directs → point terrain ou ouvrage / tronçon
- essais laboratoire → échantillon
- essais production → lot / fabrication ou échantillon selon le cas
- synthèses → fiche de synthèse

4. Parent direct du rapport :
- pour les essais bruts, le rapport est généré à partir de l'essai
- pour les synthèses, le rapport est généré à partir de la fiche de synthèse

5. Modules RaLab à prévoir à minima :
- Affaires
- Demandes
- Campagnes
- Interventions
- Points terrain
- Prélèvements
- Échantillons
- Production / lots / fabrication
- Essais laboratoire
- Essais terrain
- Synthèses techniques
- Rapports

6. Principes d'interface :
- un PMT se crée depuis une intervention terrain via un point PMT
- une granulométrie se crée depuis un échantillon
- un IM se crée depuis un échantillon issu d'un prélèvement
- une ID ne se crée pas comme essai brut mais comme fiche de synthèse alimentée par plusieurs essais sources
- un CFE se crée depuis un lot / fabrication, pas depuis un simple point terrain


48. DISTINCTION ENTRE INTERVENTION, POINT TERRAIN, PRELEVEMENT ET ECHANTILLON

Cette distinction est structurante pour la suite de RaLab.
Elle evite de melanger :
- l'action globale realisee sur le terrain,
- le point technique local ou la mesure est faite,
- l'acte de prelevement,
- et l'objet matiere suivi au laboratoire.

48.1 Intervention

L'intervention est l'action globale realisee a une date donnee, sur un chantier, une zone ou un ouvrage, avec un objectif technique donne.
C'est le conteneur operationnel.

L'intervention repond a des questions du type :
- qui est intervenu,
- quand,
- ou,
- dans quel contexte,
- avec quel objectif,
- avec quel materiel global,
- dans quelles conditions,
- pour quelle finalite technique.

Informations typiquement au niveau intervention :
- date,
- heure debut / fin,
- operateur(s),
- meteo,
- chantier / zone generale,
- section generale,
- partie d'ouvrage generale,
- objectif de mission,
- type d'intervention,
- atelier de mise en oeuvre,
- lieu de fabrication,
- date de mise en oeuvre,
- materiel d'essai global,
- mode de forage,
- sondeur,
- procede de sondage,
- observations generales.

Formule simple :
Intervention = "on va sur site pour faire une action technique".

48.2 Point terrain / point de sondage / point de controle

Le point terrain est l'objet technique elementaire situe a l'interieur d'une intervention.
C'est le point precis ou l'on mesure, observe, sonde, controle ou constate quelque chose.

Le point repond a des questions du type :
- a quel endroit exact,
- quel numero de point,
- quel repere / PK / axe / localisation precise,
- quelle profondeur / epaisseur / cote,
- quelle mesure locale,
- quelle observation locale,
- quelle conformite locale,
- quel resultat local.

Informations typiquement au niveau point :
- numero point,
- repere / PK / coordonnees,
- localisation fine,
- profondeur,
- epaisseur,
- description locale,
- resultat local,
- conclusion locale,
- photo locale,
- anomalie locale,
- conformite locale.

Formule simple :
Point = "dans cette intervention, a cet endroit precis, il s'est passe cela".

48.3 Prelevement

Le prelevement est l'acte de prise de matiere ou de collecte physique realise a partir d'un point, d'une zone ou d'un ouvrage.
Il appartient a la chaine de traçabilite matiere.

Le prelevement repond a des questions du type :
- qu'a-t-on preleve,
- ou,
- quand,
- en quelle quantite,
- selon quel mode,
- pour quel besoin,
- avec quelle destination.

Informations typiquement au niveau prelevement :
- numero prelevement,
- date / heure,
- operateur,
- point source,
- origine,
- nature apparente,
- quantite,
- contenant,
- mode de prelevement,
- reserve a reception,
- destination labo.

Formule simple :
Prelevement = "on a pris de la matiere a partir d'un point ou d'une zone".

48.4 Echantillon

L'echantillon est l'objet matiere suivi dans la suite du workflow, en particulier au laboratoire.
Il est rattache au prelevement lorsqu'il y en a un, ou cree par reception directe dans les cas prevus.

L'echantillon repond a des questions du type :
- quel code matiere est suivi,
- quelle est sa nature,
- d'ou il vient,
- a quels essais il est destine,
- dans quel etat il est recu,
- comment il est traite.

Informations typiquement au niveau echantillon :
- code echantillon,
- designation,
- nature,
- origine,
- date prelevement,
- date reception,
- quantite,
- etat a reception,
- essais demandes,
- statut labo,
- observations labo.

Formule simple :
Echantillon = "la matiere suivie et exploitee pour les essais".

48.5 Regle pratique de repartition des informations

Si l'information vaut pour toute la sortie terrain, elle doit en principe etre portee par l'intervention.
Si l'information change d'un emplacement a l'autre, elle doit en principe etre portee par le point.
Si l'information concerne l'acte de prise de matiere, elle doit etre portee par le prelevement.
Si l'information concerne la matiere recue, stockee ou essayee au laboratoire, elle doit etre portee par l'echantillon.

Exemples :
- meteo -> intervention,
- procede de sondage -> intervention,
- date de mise en oeuvre d'une couche controlee commune -> intervention,
- profondeur finale d'un sondage -> point,
- description lithologique par horizon -> point,
- EV2 d'un essai de plaque -> point,
- PMT gauche / axe / droite -> point,
- quantite de matiere prelevee -> prelevement,
- etat physique a reception labo -> echantillon,
- essais demandes -> echantillon.

48.7 Consequence de conception pour RaLab

Le systeme ne doit pas forcer tous les essais a etre crees depuis le meme parent.
La chaine amont reste commune :
Affaire -> Demande -> Campagne -> Intervention

Ensuite :
- essais terrain directs -> creation via point terrain,
- essais de sondage / controle local -> creation via point technique specifique,
- essais labo -> creation via echantillon,
- essais avec matiere prelevee -> passage obligatoire par prelevement puis echantillon,
- syntheses -> creation via fiche de synthese.

En resume :
- Intervention = mission globale,
- Point = emplacement technique local,
- Prelevement = acte de prise de matiere,
- Echantillon = matiere suivie pour les essais.


49. TABLEAU OPERATIONNEL DE MIGRATION DB

Objectif : traduire les regles de migration en instructions exploitables pour la correction de la base.

Principes :
- la migration doit toujours reconstruire une chaine metier complete,
- on cree un objet generique de structure quand un niveau manque,
- on n'invente jamais de contenu technique non justifie,
- tout objet cree automatiquement doit etre trace comme objet de migration.

Colonnes retenues :
- Cas source = situation constatee dans l'ancienne base,
- Condition SQL ou logique = condition indicative permettant de detecter le cas,
- Objet a creer = objet de structure a creer si le maillon manque,
- Table cible = table de destination ou table a enrichir dans la nouvelle base,
- Commentaire de migration = logique metier a appliquer,
- Niveau de confiance = niveau de fiabilite de la migration automatique.

Pour la migration, les objets generiques suivants doivent pouvoir etre crees :
- affaire generique de migration,
- demande generique de migration,
- campagne generique de migration,
- intervention generique de migration,
- point terrain generique,
- point specialise generique (PMT, sondage, carottage, forage, penetrometre, etc.),
- prelevement generique,
- echantillon generique,
- lot generique de fabrication,
- fiche synthese generique,
- rapport generique de rattachement.

49.2 Champs transverses de traçabilite migration

Tout objet cree ou modifie par migration doit pouvoir porter :
- migration_created = true / false,
- migration_reason,
- legacy_table,
- legacy_uid,
- legacy_reference si disponible,
- migration_confidence = high / medium / low,
- review_required = true / false,
- migration_batch.

49.3 Regle mere a conserver

Quand une information de rattachement manque, on cree un objet generique de structure afin de preserver la chaine metier cible, sans inventer de contenu technique non justifie.
